# Responsibilities

## app

Application routes.

Layouts.

Metadata.

Pages.

Server Components.

---

## components

Reusable UI components.

Should contain no business logic.

---

## content

Markdown and MDX content.

Acts as the portfolio database.

---

## config

Site configuration.

Navigation.

Metadata.

SEO.

Theme configuration.

---

## constants

Static values.

Colors.

Animation durations.

Breakpoints.

Social links.

---

## services

External integrations.

Email.

GitHub.

Analytics.

Future APIs.

---

## lib

Reusable helper libraries.

Markdown parsing.

Utilities.

Content loaders.

---

## hooks

Custom React hooks.

---

## public

Images.

Icons.

Fonts.

Favicons.

Open Graph images.

Shaders.

---

## styles

Global CSS.

Tailwind layers.

Variables.

---

## types

Shared TypeScript interfaces.

---

## utils

Pure helper functions.

Formatting.

Validation.

Date utilities.

Slug generation.

---

# Rule

Every folder should have one clear responsibility.
