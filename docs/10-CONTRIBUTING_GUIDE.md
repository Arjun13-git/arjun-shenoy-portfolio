# Contribution Guide

Welcome to Neural Horizon.

This project is designed as a long-term engineering portfolio with a strong emphasis on maintainability, scalability, accessibility, and performance.

---

# Core Principles

Performance over Flash

Content over Decoration

Clarity over Complexity

Architecture over Shortcuts

Timeless over Trendy

Meaningful Interactions over Unnecessary Animations

---

# Folder Responsibilities

app/

Application routing.

components/

Reusable UI components.

content/

Markdown and MDX content.

architecture/

Technical documentation.

design/

Visual assets and design references.

docs/

Project documentation.

lib/

Shared utilities.

hooks/

React hooks.

styles/

Global styling.

public/

Static assets.

---

# Development Rules

Prefer composition over duplication.

Keep components reusable.

Avoid deeply nested components.

Keep business logic outside UI.

Use TypeScript strictly.

Use semantic HTML.

Prioritize accessibility.

---

# Component Rules

Every component should

- Have a single responsibility.
- Be reusable.
- Accept typed props.
- Avoid unnecessary state.

---

# Styling Rules

Use TailwindCSS.

Avoid inline styles.

Maintain consistent spacing.

Use design tokens.

---

# Performance Rules

Avoid unnecessary client components.

Use lazy loading.

Optimize images.

Keep bundle size small.

Measure before optimizing.

# Pull Request Checklist

- Builds successfully

- Responsive

- Accessible

- Performance maintained
